# ChainTrust

블록체인 기반 중고거래 이력 인증 서비스

## 1. 프로젝트 소개

ChainTrust는 중고거래에서 물품의 소유권 이전 이력을 블록체인에 기록하여,
거래 내역의 신뢰성을 높이는 웹 서비스입니다.

사용자는 MetaMask 지갑을 연결해 물품을 등록하고,
거래 후 소유권을 다른 지갑 주소로 이전할 수 있습니다.
등록 및 이전 기록은 Ethereum Sepolia 테스트넷에 남아,
이후 물품의 거래 이력을 확인할 수 있습니다.

## 2. 개발 배경

중고거래에서는 판매자가 실제 소유자인지,
이전에 어떤 거래 이력이 있었는지 확인하기 어렵다는 문제가 있습니다.

ChainTrust는 블록체인의 변경이 어려운 기록 특성을 활용해,
거래 이력을 투명하게 확인할 수 있는 구조를 실험하기 위해 개발했습니다.

## 2-1. 개발 과정

이 프로젝트는 AI 기반 개발 도구를 적극 활용하여 구현한 블록체인 웹 서비스입니다.  
기획, 화면 설계, 코드 초안 작성, 오류 해결 과정에서 AI의 도움을 받았으며,
저는 서비스 구조 설계, UI 구성, 기능 테스트, MetaMask 연동 확인, 배포 및 발표 자료 정리를 담당했습니다.

## 3. 주요 기능

- MetaMask 지갑 연결 및 계정 기반 사용자 식별
- 물품 등록
- 전체 물품 목록 조회
- 물품 상세 정보 조회

## 6. 핵심 구현 내용

### MetaMask 지갑 연동

사용자의 Ethereum 지갑 주소를 기반으로 로그인 상태를 관리했습니다.

지갑 연결, 연결 해제, 계정 변경을 감지하여 현재 선택된 계정을 기준으로 기능을 사용할 수 있도록 구현했습니다.

### 블록체인 연동

Ethereum Sepolia 테스트넷에 스마트 컨트랙트를 배포하고, MetaMask를 통해 물품 등록 및 소유권 이전 트랜잭션을 수행할 수 있도록 구현했습니다.

사용자가 수행한 작업은 블록체인에 기록되며 거래 이력 화면에서 확인할 수 있습니다.

### 소유권 이전 및 이력 관리

현재 소유자만 물품의 소유권을 이전할 수 있도록 구현했습니다.

소유권이 이전되면 새로운 소유자 정보와 함께 이전 기록이 저장되며, 사용자는 물품별 거래 이력을 확인할 수 있습니다.



## 7. 트러블 슈팅

### 1. MetaMask 계정 전환 반영 문제

MetaMask에서 계정을 변경해도 앱의 현재 지갑 주소가 즉시 갱신되지 않는 문제가 있었습니다.

이를 해결하기 위해 MetaMask의 계정 변경 이벤트를 감지하여 현재 선택된 계정 주소가 앱 상태에 반영되도록 수정했습니다.

### 2. Sepolia 네트워크 판별 오류

MetaMask가 Sepolia 네트워크에 연결되어 있음에도 홈 화면에서 `Wrong Network`가 표시되는 문제가 있었습니다.

원인을 분석한 결과 네트워크 ID(chainId) 처리 방식의 차이로 인해 발생한 문제였으며, 네트워크 정보를 일관된 형식으로 처리하도록 수정하여 해결했습니다.

## 8. 한계점 및 개선 방향

### 한계점

* 하나의 사용자가 여러 지갑 주소를 생성할 수 있어 거래 이력을 인위적으로 만들 수 있음
* 판매자에 대한 신뢰도나 평판 정보를 확인할 수 있는 기능이 부족함
* 거래 이력은 확인할 수 있지만 판매자 프로필을 통한 종합적인 이력 조회 기능은 제공하지 않음

### 개선 방향

* 판매자 프로필 및 거래 이력 조회 기능 추가
* 거래 완료 후 리뷰 및 평판 시스템 추가
* 물품 가격 정보와 블록체인 가스비를 분리하여 원화(KRW) 기반 가격 표시 지원
* IPFS 등을 활용한 이미지 및 메타데이터 관리 방식 개선

## 9. 배운 점

이번 프로젝트를 통해 React 기반 웹 서비스와 MetaMask, 스마트 컨트랙트를 연결하는 전체 흐름을 경험했습니다.

특히 블록체인은 단순히 데이터를 저장하는 기술이 아니라, 수정하기 어려운 거래 기록을 남긴다는 점에서 신뢰가 중요한 서비스와 잘 맞는다는 것을 배웠습니다.

또한 ChatGPT, Codex, Stitch 등의 AI 도구를 활용하여 기획, UI 설계, 문서화, 코드 구현 및 오류 해결 과정을 반복하면서 개발 생산성을 높이는 경험을 할 수 있었습니다.

AI가 제안한 코드를 그대로 사용하는 것이 아니라, 실제 동작을 확인하고 문제를 수정하는 과정을 통해 새로운 기술을 빠르게 학습하고 적용하는 방법을 익힐 수 있었습니다.
